## 📚 O que alunos de 7° ano conhecem até o meio do semestre

Com base na BNCC (EF07MA01–EF07MA14) e no currículo SESI/1° semestre:

### 1° Bimestre (já estudado com profundidade)
- Múltiplos e divisores, MMC e MDC, números primos
- Frações: leitura, simplificação, equivalência, comparação
- Operações com frações: soma, subtração, multiplicação, divisão
- Números decimais: leitura, comparação, operações
- Porcentagem: cálculo simples, acréscimos e decréscimos

### 2° Bimestre (estudando agora / até o meio do semestre)
- Números inteiros: positivos e negativos, reta numérica
- Operações com inteiros: adição, subtração (temperatura, débito)
- Introdução à ideia de variável (pré-álgebra, sem equações formais)
- Proporcionalidade direta: reconhecer e calcular grandezas
- Regra de três simples (contextos práticos)

### O que NÃO entra (ainda não viram)
- ❌ Equações do 1° grau formais (2x + 5 = 25) — vem no 2° semestre
- ❌ Geometria (áreas, volumes, ângulos, Pitágoras) — 3° bimestre
- ❌ Estatística e probabilidade — 4° bimestre
- ❌ Potências, raízes, logaritmos — não é do 7° ano
- ❌ Funções, trigonometria, matrizes — Ensino Médio

## ⚠️ Regras de Design Pedagógico — O que nunca pode aparecer

```
PROIBIDO EM QUALQUER FASE:
  ❌  Equações com variável isolada (2x + 5 = 25) — não viram ainda
  ❌  Variáveis nos dois lados da equação
  ❌  Ângulos, graus, reflexão, geometria plana ou espacial
  ❌  Probabilidade e estatística (média, moda, mediana)
  ❌  Potências (3²) ou raízes quadradas
  ❌  Logaritmos, funções, trigonometria, matrizes
  ❌  Conteúdos do 8°, 9° ano ou Ensino Médio
  ❌  Linguagem adulta, sarcasmo, ironia ou temas violentos
  ❌  Tempo de leitura longo (máx. 2 linhas de enunciado por desafio)

OBRIGATÓRIO EM TODA FASE:
  ✅  Enunciado curto, direto, com no máximo 2 linhas
  ✅  Visual que ILUSTRA o problema (o aluno vê antes de ler)
  ✅  Números pequenos e concretos (nada de 4.872 ÷ 0,031)
  ✅  Contexto real do cotidiano (comida, tempo, dinheiro, temperatura)
  ✅  Numpad visual (crianças de 11 anos digitam mal em teclado)
  ✅  Pista gratuita na fase 1 de cada sala (a mais fácil)
  ✅  Feedback positivo mesmo no erro: "Quase! Veja a dica."
  ✅  Animação que MOSTRA a resposta correta após 3 erros
```

## 🤖 Prompts Prontos para IA de Código

### Prompt A — Fase de Frações com Barras Visuais

```
Atue como desenvolvedor React especialista em jogos educativos para crianças de 11 anos.

Crie o componente Phase6_Caverna_Adicao.jsx para a Sala 2, Fase 6.

Tema: adição de frações com mesmo denominador.
Desafio: "Você mistura 2/7 de poção azul com 3/7 de poção verde. Quanto de poção você tem no total?"

Requisitos visuais:
- Dois frascos animados lado a lado (SVG), um azul (2/7 cheio) e um verde (3/7 cheio)
- Setas animadas dos dois frascos para um terceiro frasco vazio no centro
- Frasco central vai se enchendo conforme o aluno digita a resposta
- Campo de entrada com dois sub-campos: numerador e denominador separados por traço visual
- Numpad visual com dígitos 0–9 e tecla de apagar (crianças de 11 anos)
- Se correto (5/7): os frascos explodem em bolhas coloridas + sons graves e agudos
- Se incorreto: frascos tremem + mensagem "Quase! O denominador fica igual (7). Some só os numeradores."
- Pista GRATUITA (sem custo de pontos): aparece ao clicar em "Preciso de ajuda?"
- Props: onPhaseComplete(points, timeBonus), timeLimit=60, attempts=3
- Três variações aleatórias: 2/7+3/7=5/7 | 1/5+3/5=4/5 | 3/9+4/9=7/9
```

### Prompt B — Fase de Inteiros com Blocos Visuais

```
Atue como desenvolvedor React para jogos educativos.

Crie o componente Phase4_Castelo_Inteiros.jsx para a Sala 4, Fase 4.

Tema: adição de inteiros com sinais diferentes.
Desafio: "(+7) + (−4) = ?"

Requisitos visuais:
- Dois conjuntos de blocos na tela: 7 blocos AZUIS (positivos) e 4 blocos VERMELHOS (negativos)
- Animação de "cancelamento": cada bloco vermelho apaga um azul (aparecem raios entre eles)
- Os 3 blocos azuis restantes ficam acesos — esses representam a resposta
- Campo de input simples para o aluno confirmar o valor (+3)
- Botão de sinal: aluno escolhe se a resposta é positiva ou negativa (dois botões grandes: [+] e [−])
- Feedback correto: blocos voam para formar o número na tela
- Feedback incorreto: blocos voltam à posição original + dica: "Subtraia os menores dos maiores"
- Três variações: (+7)+(−4)=+3 | (+5)+(−8)=−3 | (+10)+(−6)=+4
```

### Prompt C — Fase de Porcentagem com Barra Visual

```
Atue como desenvolvedor React para jogos educativos infantis.

Crie o componente Phase7_Porto_Desconto.jsx para a Sala 3, Fase 7.

Tema: porcentagem simples — calcular 10% de um valor.
Desafio: "Um produto custa R$ 80,00 e tem 10% de desconto. Qual o valor do desconto em reais?"

Requisitos visuais:
- Barra horizontal dividida em 10 partes iguais (cada parte = 10%)
- Uma parte fica destacada em amarelo dourado representando o "desconto"
- Preço do produto exibido em placa de mercado animada
- Campo de entrada grande com R$ fixo antes (sem ter que digitar o símbolo)
- Numpad visual com vírgula para decimais
- Ao acertar (R$ 8,00): chuva de moedas douradas + placa com "Você economizou!"
- Ao errar: parte da barra pisca em vermelho + dica: "10% = dividir por 10"
- Três variações: 10% de 80=8 | 10% de 120=12 | 10% de 50=5
- Variação extra para alunos mais rápidos: 50% de 60=30 (aparece apenas se resolver em menos de 15s)
```
